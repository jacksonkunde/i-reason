class ireason.iadd.addition_config.FillZerosConfig(apply_consistently, fill=None, probability=None)[source]¶

Bases: object

Configuration for zero padding in numbers.

apply_consistently¶

Determines if zero padding is applied uniformly across the dataset. - If True, the padding behavior is consistent throughout the dataset. - If False, padding is applied based on a specified probability.

Type:

bool

fill¶

Specifies whether to apply padding when apply_consistently is True. - If True, numbers are padded with leading zeros to match the length of the longest term in each problem. - If False, padding is not applied. - Required if apply_consistently is True.

Type:

Optional[bool]

probability¶

The probability of applying padding when apply_consistently is False. - A float between 0 and 1 representing the chance of padding each addition problem. - Required if apply_consistently is False.

Type:

Optional[float]

Examples

Consistent Padding:

apply_consistently: True fill: True

Random Padding:

apply_consistently: False probability: 0.5

Notes

  • When padding is applied, leading zeros are added so that all terms in the problem have the same length.

  • Leading zeros are not counted when determining the number of digits in a term.

class ireason.iadd.addition_config.HoldOutDigits(digits, positions, apply_to_training_set, include_in_test_set)[source]¶

Bases: object

Specifies digits to exclude from the training set and include in the test set at certain positions.

digits¶

List of digits to hold out (integers between 0 and 9).

Type:

List[int]

positions¶

List of positions where the digits are held out. - Positions are counted from the right (least significant digit). - Position 1 refers to the units place.

Type:

List[int]

apply_to_training_set¶

Determines if the hold-out is applied to the training set. - If True, the specified digits at the specified positions are excluded from the training set.

Type:

bool

include_in_test_set¶

Determines if the held-out digits are included in the test set. - If True, the test set will include problems that have the specified digits at the specified positions.

Type:

bool

Example

hold_out_digits:

digits: [7, 8] positions: [1, 2] apply_to_training_set: True include_in_test_set: True

Notes

  • Digits: Must be integers between 0 and 9.

  • Positions: Must be positive integers.
